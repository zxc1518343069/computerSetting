# TestConfigModal UI 重构方案：性能实验室 2.0 (Cyber-Refined)

## 1. 设计愿景 (Design Vision)

将 `TestConfigModal` 从一个简单的“数据对比弹窗”重塑为一个**“沉浸式性能分析仪表盘”**。

- **核心风格**：赛博精炼 (Cyber-Refined) —— 结合深邃的背景、高饱和度的性能指示色、以及精致的毛玻璃质感。
- **视觉关键词**：通透感、仪表化、动态反馈、专业度。

---

## 2. 核心改进点 (Key Enhancements)

### 2.1 布局与容器 (Layout & Container)

- **沉浸式框架**：移除 Modal 默认的边距，采用全屏/大尺寸的通透布局。
- **动态背景**：在弹窗背景中使用微弱的渐变光晕（Mesh Gradient），随选中的配置性能等级（入门/中端/旗舰）变换色调。
- **毛玻璃侧边栏/顶栏**：使用 `backdrop-blur-xl` 增强层次感。

### 2.2 交互式配置选择 (Config Selection) - [重构重点]

- **从“标签”到“方案卡片”**：彻底弃用微小的 Tag，改为响应式的**方案网格 (Config Grid)**。
    - **卡片内容**：包含方案全名、核心硬件摘要（CPU + GPU）、以及性能等级图标。
    - **视觉反馈**：选中态增加 `ring-2` 环绕发光效果，并应用方案专属主题色。
- **布局优化**：将选择区独立为一个带背景色的区块，确保在多选时依然有良好的视觉引导。

### 2.3 性能可视化 (Performance Visualization)

- **性能段位图例 (Legend)**：在显著位置增加 FPS 颜色含义说明：
    - 🟢 **电竞级 (144+)**：极致流畅
    - 🔵 **流畅级 (60-144)**：主流标准
    - 🟠 **及格级 (30-60)**：基本可玩
    - 🔴 **吃力级 (<30)**：建议降画质
- **FPS 增强展示**：保留字符串，但在高帧率数值旁增加微弱的闪烁或发光效果，增强“高性能”的心理暗示。

### 2.4 视图逻辑：深度融合 (Unified Battle View)

- **拒绝频繁切换**：将性能对比与核心硬件参数合并展示。
- **对比矩阵**：在每个游戏的对比项中，直接显示：`[方案名] | [CPU+GPU] | [FPS数值]`。
- **差异高亮**：如果两个方案的 CPU 相同但 GPU 不同，自动淡化 CPU 信息，高亮 GPU 差异，引导用户关注变量。

### 2.5 结构修正 (Structural Fixes)

- **Header 重排**：将功能切换按钮下移或左移，为 Modal 右上角的关闭按钮留出至少 80px 的安全区域，彻底解决重合问题。
- **滚动优化**：方案选择区固定在顶部，仅下方对比内容滚动。

### 2.6 黑夜模式适配策略 (Dark Mode Strategy)

- **深色美学**：黑夜模式下采用 `slate-950` 作为基色，配合 `backdrop-blur` 营造深邃的玻璃质感。
- **霓虹引导**：利用 Tailwind 的 `dark:shadow-[0_0_15px_rgba(59,130,246,0.3)]` 为选中项增加霓虹发光效果，强化视觉焦点。
- **色彩校准**：针对深色背景微调 FPS 颜色（如将 Emerald-500 调整为更亮的 Emerald-400），确保对比度。

---

## 3. 实施步骤 (Implementation Steps)

1. **Step 1: 结构重组** - 调整 Modal Header 布局，解决关闭按钮重合问题，预留图例空间。
2. **Step 2: 方案卡片开发** - 实现大尺寸方案选择卡片，集成硬件摘要展示。
3. **Step 3: 视图融合** - 重构内容区，实现“游戏-方案-硬件-性能”四位一体的对比矩阵。
4. **Step 4: 动效与细节** - 注入 `framer-motion` 动画，完善黑夜模式下的光影细节。

---

## 4. 视觉规范 (Visual Specs)

- **色彩方案**：
    - 基础：`slate-950` (背景), `white/10` (边框)
    - 性能：`emerald-500` (144+ FPS), `blue-500` (60+ FPS), `amber-500` (30+ FPS)
    - 装饰：`indigo-500` 渐变到 `purple-500`
- **圆角**：统一使用 `rounded-3xl` (24px) 和 `rounded-2xl` (16px)。
- **字体**：数字部分强制使用 `font-mono` 增强专业感。

---

## 5. 待确认事项

1. **动画性能**：是否允许引入 `framer-motion`？（推荐，可显著提升质感）
2. **对比上限**：目前支持多选，是否需要限制最大对比数量（如 3 个）以保证 UI 整洁？
3. **数据源**：FPS 估算逻辑是否需要进一步细化（如加入内存频率影响）？
